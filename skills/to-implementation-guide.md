# Skill: to-implementation-guide

Convert a generic principle into a project-specific implementation guide.

## Inputs

- Generic principle path
- Project context
- Relevant modules, boundaries, adapters, or workflows
- Target guide location

## Output

A practical implementation guide for the target project.

## Procedure

1. Read the generic principle and metadata.
2. Identify the expected implementation flow for the project.
3. Add examples or pseudocode using project terminology.
4. Document edge cases and anti-patterns.
5. Add review notes for humans and agents.

## Constraints

- Do not tie the guide to unrelated frameworks or vendors.
- Preserve the architectural reasoning behind the principle.
- Keep examples practical but not overly narrow.
